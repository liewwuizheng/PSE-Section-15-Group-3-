## 3. Kanban List Definitions
To maintain a high standard of Completeness (DoD), our Kanban boards are organized according to the following automated columns:

* ** Product Backlog:** A complete list of 25 features to be developed (Functional Requirement 1 - Functional Requirement 25).
* ** Sprint [x] - To Do:** Selected features for the currect week's work.
* ** Sprint [x] - Doing:** Tasks currently being worked on by the responsible role.
* ** Sprint [x] - Done:** Features that have passed QA and met all the acceptance criteria.

---

## 4. Trello Card Structure Analysis
Every card on our Kanban board follows a standardized format to ensure all team members can clearly understand it:

| Field | Description |
| :--- | :--- |
| User Story | Clear statement of value (e.g. "As a patient, I want...") |
| Acceptance Criteria | A checklist of specific requirements to consider the task "Done". |
| Priority | Tagged each task as 'High', 'Medium' or 'Low'. |
| Responsible Role | Assigned to the Prorotype Lead, UI/UX Designer, or Testing Lead. |
| Definition of Done | A checklist prepared for each task to ensure that the task is done completely. |
